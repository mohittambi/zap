## Role purposes (database names)

| DB role | Business name | Purpose |
| ------- | ------------- | ------- |
| `admin` | Administrator | Full access via `*:*` wildcard; user management; audit; all modules; GRN audit approval |
| `ops_manager` | Operations Manager | Inbound/outbound ops, vendors, warehouses, bins, bulk import/export, consignments, forms, dashboard queries, vendor delete |
| `warehouse_staff` | Warehouse Staff | GRNs, bin locations, picking, warehouse workflows, operational forms |
| `finance` | Finance / Accounts | PO/GRN commercial visibility, debit/credit notes, vendors (read), forms, dashboard queries |
| `merchandising` | Merchandising / Category | Listings, catalogues, labels, secondary listings, focus lists |
| `sales` | Sales / Key Accounts | Read-focused catalogue, listings, outbound visibility, dashboard queries |

Assign roles in **Settings ‚Üí Users** (admin only). Users may hold multiple roles; effective permissions are the union of all assigned roles.

## Admin user management (Settings ‚Üí Users)

**URL:** `/settings/users`
**Required permission:** `admin` role with `*:*` (wildcard)

### List users
- GET `/api/admin/users` ‚ÄĒ returns email, roles, active status, last login.

### Create user
1. Open **Settings ‚Üí Users** ‚Üí **Add user**.
2. Enter email, temporary password, and one or more roles.
3. POST `/api/admin/users` ‚ÄĒ user can log in immediately.

Or via script for bulk admin bootstrap:

```bash
cd web
export DATABASE_URL="<prod session pooler :5432>"
node scripts/seed-prod-admin-users.mjs
# Optional: node scripts/seed-prod-admin-users.mjs --email someone@ecraftindia.com
```

### Reset password
1. **Settings ‚Üí Users** ‚Üí select user ‚Üí **Reset password**.
2. PATCH `/api/admin/users/{id}` with `{ "password": "..." }`.
3. Sets `token_invalidated_at` ‚ÄĒ existing sessions are invalidated on next request.

### Activate / deactivate
- PATCH `/api/admin/users/{id}` with `{ "isActive": true|false }`.
- Deactivated users cannot log in; history remains in audit logs.

### Change roles
- PATCH `/api/admin/users/{id}` with `{ "roleIds": [1, 2] }`.
- Changes apply on next login (or immediately for new API calls after permission cache refresh).

### Audit
- Admin actions (create user, role change, password reset, deactivate) are written to `admin_audit_log` via `adminAuditService`.

## Apply on production

```bash
cd web
export DATABASE_URL="$(grep '^DATABASE_URL_DIRECT=' .env.production.local | cut -d= -f2-)"

# 1. Migration 073 (idempotent)
bash scripts/run_migrations.sh
# Or apply only 073:
# psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -f migrations/073_rbac_assign_catalogued_permissions.sql

# 2. Seed Saumya + Ankit admins (prints passwords once)
node scripts/seed-prod-admin-users.mjs

# 3. Verify
psql "$DATABASE_URL" -c "
SELECT u.email, array_agg(r.name ORDER BY r.name) roles
FROM users u
LEFT JOIN user_roles ur ON ur.user_id = u.id
LEFT JOIN roles r ON r.id = ur.role_id
GROUP BY u.email ORDER BY u.email;
"
psql "$DATABASE_URL" -c "
SELECT r.name role, p.resource || ':' || p.action perm
FROM role_permissions rp
JOIN roles r ON r.id = rp.role_id
JOIN permissions p ON p.id = rp.permission_id
WHERE p.resource IN ('bins','forms','query_builder','vendors')
  AND p.action IN ('manage','write','read','delete')
ORDER BY 1, 2;
"
```

**Safety:** `seed-prod-admin-users.mjs` refuses to run unless `DATABASE_URL` contains prod project ref `bxgmcddxmlsgrflnbywv`.

## Creating users for other roles

Do **not** seed demo users on production. For each new team member:

1. Admin creates account in **Settings ‚Üí Users** with the appropriate role(s).
2. Share temporary password securely (not email ‚ÄĒ Zap does not send invite emails yet).
3. User logs in at https://zap-rust.vercel.app/login and should change password (admin can set a new one anytime).

Suggested first production assignments (when people are ready):

| Person / team | Suggested role |
| ------------- | -------------- |
| Warehouse lead | `warehouse_staff` |
| Ops lead | `ops_manager` |
| Accounts | `finance` |
| Category / listings | `merchandising` |
| Sales / KAM | `sales` |
